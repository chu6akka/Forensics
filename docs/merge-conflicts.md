# Как выбирать Current/Incoming при конфликте в GitHub

Когда GitHub показывает конфликт с вариантами **Current** и **Incoming**, это означает:

- **Current (Current change)** — версия из вашей текущей ветки (то, что есть у вас сейчас).
- **Incoming (Incoming change)** — версия из ветки, которую вы пытаетесь влить (обычно PR/ветка-источник).
- **Both (если есть кнопка)** — оставляет оба варианта рядом; затем вы вручную приводите итог к нужному виду.

## Как понять, что выбирать

1. Посмотрите, какие правки должны остаться в итоговой версии файла.
2. Если нужно сохранить вашу правку — выбирайте **Current**.
3. Если нужно заменить вашей правкой изменения из другой ветки — выбирайте **Incoming**.
4. Если нужны оба изменения — выбирайте **Both**, а затем отредактируйте итоговый текст вручную.

## Практический совет

- Всегда читайте контекст вокруг конфликта: иногда обе версии частично верны, и правильный ответ — объединить.
- После выбора варианта убедитесь, что итоговый файл логически корректен и проект собирается.
